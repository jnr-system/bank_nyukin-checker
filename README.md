# bank_nyukin-checker

入金データと楽楽販売の顧客データを自動照合するツールです。

Googleスプレッドシートに記録された振込名義・支払者氏名を読み取り、楽楽販売の顧客データと照合して、一致した場合にスプレッドシート・楽楽販売のステータス更新を行います。スプレッドシートごとに照合モードや後続処理（SMS送信など）を個別に設定できます。

## 機能（共通）

- Googleスプレッドシートから未照合の入金データを取得（最大4シートまで同時処理）
- 楽楽販売APIから顧客データ（最大5,000件）を一括取得
- 重複レコード（`進捗` 列に「重複」を含む）を自動除外
- 照合結果をスプレッドシートA列に書き込み（`要確認` は赤文字で表示）
- 照合済みレコードの楽楽販売フラグ（成約管理DB・問合せ管理DB）と入金日を更新

## スプレッドシートごとの処理内容

| シート | 照合モード | 楽楽販売更新 | SMS送信 |
|--------|-----------|-------------|---------|
| SPREADSHEET_ID_1（楽天第二） | カナ照合 | ✅ | ✅ |
| SPREADSHEET_ID_2（電算・コンビニ払い） | 漢字照合 | ✅ | ❌ |
| SPREADSHEET_ID_3（JACCSローン） | 漢字照合 | ✅ | ❌ |

## 照合フロー

### シート1：カナ照合モード（`kana`）

```
銀行振込名義
    │
    ▼
スキップ判定（法人・手数料・銀行振込等）→ 要確認
    │
    ▼
振込識別番号照合（口座名義の先頭数字 ↔ 楽楽の振込識別番号）
    ├─ 一致 → 照合済み
    └─ 番号なし or 不一致 → カナ照合へ
        │
        ├─ 第1段階：正規化完全一致
        │       └─ 一致 → 照合済み
        │
        └─ 第2段階：先頭文字グルーピング → Gemini AI照合
                ├─ 一致 → 照合済み
                ├─ 候補なし → 要確認（フリカナ未登録）
                └─ 不一致 → 要確認
```

照合後：楽楽販売の入金日・照合済みフラグを更新 → SMS送信

### シート2・3：漢字照合モード（`kanji`）

```
支払者氏名（漢字）
    │
    ▼
第1段階：漢字氏名の完全一致（楽楽の「請求先お名前」と突合）
    ├─ 一致 → 照合済み
    └─ 不一致 → Gemini AI照合へ
            ├─ 一致 → 照合済み
            └─ 不一致 → 要確認
```

照合後：楽楽販売の入金日・照合済みフラグを更新（SMS送信なし）

## セットアップ

### 1. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. 環境変数の設定

`env` ディレクトリに `.env` を作成して各値を設定します。

| 変数名 | 説明 |
|--------|------|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GEMINI_API_KEY` | Google AI Studio で発行したGemini APIキー |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GoogleサービスアカウントのJSONを1行で記載 |
| `ACCRETE_ACCOUNT_ID` | accreteのアカウントID |
| `ACCRETE_REQUEST_ID` | accreteのリクエストID |
| `ACCRETE_PASSWORD` | accreteのパスワード |

### スプレッドシートごとの設定（最大3つまで）
| 変数名 | 説明 |
|--------|------|
| `SPREADSHEET_ID_1` | 1つ目のスプレッドシートID |
| `SPREADSHEET_MATCH_MODE_1` | 照合モード（`kana` または `kanji`） |
| `SPREADSHEET_NAME_COL_1` | 口座名義・氏名の列（例：`C`） |
| `SPREADSHEET_DATE_COL_1` | 入金日の列（例：`D`） |
| `SPREADSHEET_AMOUNT_COL_1` | 入金金額の列（例：`E`） |
| `SPREADSHEET_SMS_1` | SMS送信を行うか（`true` または `false`） |

※ `_2`, `_3` についても同様に設定可能。旧来の `SPREADSHEET_ID` のみの指定も後方互換性で動作します（SMS送信あり・カナ照合モードとして扱われます）。

シート名は `YYYYMM`（例: `202604`）で、以下の列構成を想定しています（列位置は環境変数で指定）。

| 列 | 内容 |
|----|------|
| A | 照合結果（ツールが書き込む、固定） |
| 指定 | 入金日 |
| 指定 | 入金金額 |
| 指定 | 銀行口座名義（照合対象） |

## 使い方

```bash
# 当月シートで本番実行
python main.py

# 指定月シートで実行
python main.py --sheet 202603

# ドライラン（スプシ・楽楽・SMSへの書き込みなし）
python main.py --dry-run

# スプシのみ更新（楽楽フラグ更新・SMS送信はスキップ）
python main.py --sheet-only

# 組み合わせ
python main.py --sheet 202603 --dry-run
```

## スプレッドシートの書き込み値

| 値 | 意味 |
|----|------|
| `照合済み（手配番号）` | 照合成功 |
| `照合済み（楽楽確認済み）` | 楽楽側で既に照合済みのためスキップ |
| `要確認`（赤文字） | 照合失敗・スキップ対象（法人等） |
| `要確認（金額不一致）`（赤文字） | 口座名義は一致したが金額が不一致 |
| `要確認（フリカナ未登録）`（赤文字） | 楽楽にフリカナ未登録のため照合不可 |
| `要確認（エラー）`（赤文字） | 照合処理中にエラー発生 |

## ファイル構成

```
bank_nyukin-checker/
├── main.py          # エントリポイント・照合ループ
├── rakuraku.py      # 楽楽販売API（データ取得・フラグ更新）
├── matching.py      # カナ正規化・振込識別番号照合・Gemini照合ロジック
├── spreadsheet.py   # Googleスプレッドシート読み書き
├── sms.py           # accrete SMS送信API連携
├── requirements.txt
└── logs/            # 実行ログ（自動生成）
```

ログは `logs/matching.log` にも保存されます。
