# bank_nyukin-checker

銀行振込入金データと楽楽販売の顧客データを自動照合するツールです。

Googleスプレッドシートに記録された銀行振込の口座名義を読み取り、楽楽販売の顧客データと照合して、一致した場合にスプレッドシート・楽楽販売のステータス更新およびSMS送信を行います。

## 機能

- Googleスプレッドシートから未照合の振込データを取得
- 楽楽販売APIから顧客データ（最大5,000件）を一括取得
- 重複レコード（`進捗` 列に「重複」を含む）を自動除外
- 3段階の照合ロジック：
  1. **振込識別番号照合**：口座名義の先頭番号と楽楽の振込識別番号を突合
  2. **カナ完全一致**：正規化後のカナ名を完全一致で照合
  3. **Gemini AIファジー照合**：先頭文字グループ内でAIが同一人物を判定
- 照合結果をスプレッドシートB列に書き込み（`要確認` は赤文字で表示）
- 照合済みレコードの楽楽販売フラグ（成約管理DB・問合せ管理DB）を「照合済み」に更新
- 照合済み顧客へSMS送信（accreteのSMS API使用）

## 照合フロー

```
銀行振込名義（D列）
    │
    ▼
スキップ判定（法人・手数料・銀行振込等）→ 要確認
    │
    ▼
振込識別番号照合（口座名義の先頭数字 ↔ 楽楽の振込識別番号）
    ├─ 一致 → 照合済み
    └─ 番号なし or 不一致 → カナ照合へ
        │
        ├─ 第1段階：正規化完全一致
        │       └─ 一致 → 照合済み
        │
        └─ 第2段階：先頭文字グルーピング → Gemini AI照合
                ├─ 一致 → 照合済み
                ├─ 候補なし → 要確認（フリカナ未登録）
                └─ 不一致 → 要確認
```

## セットアップ

### 1. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. 環境変数の設定

`env` ディレクトリに `.env` を作成して各値を設定します。

| 変数名 | 説明 |
|--------|------|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン |
| `GEMINI_API_KEY` | Google AI Studio で発行したGemini APIキー |
| `SPREADSHEET_ID` | 対象GoogleスプレッドシートのID |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GoogleサービスアカウントのJSONを1行で記載 |
| `ACCRETE_ACCOUNT_ID` | accreteのアカウントID |
| `ACCRETE_REQUEST_ID` | accreteのリクエストID |
| `ACCRETE_PASSWORD` | accreteのパスワード |

### 3. スプレッドシートの形式

シート名は `YYYYMM`（例: `202604`）で、以下の列構成を想定しています。

| 列 | 内容 |
|----|------|
| B | 照合結果（ツールが書き込む） |
| C | 入金日 |
| D | 銀行口座名義（照合対象） |

## 使い方

```bash
# 当月シートで本番実行
python main.py

# 指定月シートで実行
python main.py --sheet 202603

# ドライラン（スプシ・楽楽・SMSへの書き込みなし）
python main.py --dry-run

# スプシのみ更新（楽楽フラグ更新・SMS送信はスキップ）
python main.py --sheet-only

# 組み合わせ
python main.py --sheet 202603 --dry-run
```

## スプレッドシートの書き込み値

| 値 | 意味 |
|----|------|
| `照合済み（手配番号）` | 照合成功 |
| `照合済み（楽楽確認済み）` | 楽楽側で既に照合済みのためスキップ |
| `要確認`（赤文字） | 照合失敗・スキップ対象（法人等） |
| `要確認（フリカナ未登録）`（赤文字） | 楽楽にフリカナ未登録のため照合不可 |
| `要確認（エラー）`（赤文字） | 照合処理中にエラー発生 |

## ファイル構成

```
bank_nyukin-checker/
├── main.py          # エントリポイント・照合ループ
├── rakuraku.py      # 楽楽販売API（データ取得・フラグ更新）
├── matching.py      # カナ正規化・振込識別番号照合・Gemini照合ロジック
├── spreadsheet.py   # Googleスプレッドシート読み書き
├── sms.py           # accrete SMS送信API連携
├── requirements.txt
└── logs/            # 実行ログ（自動生成）
```

ログは `logs/matching.log` にも保存されます。
