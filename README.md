# bank_nyukin-checker

銀行振込入金データと楽楽販売の顧客データを自動照合するツールです。

Googleスプレッドシートに記録された銀行振込の口座名義を読み取り、楽楽販売の顧客フリカナと照合して、一致した場合に双方のステータスを自動更新します。

## 機能

- Googleスプレッドシートから未照合の振込データを取得
- 楽楽販売APIから顧客データ（最大5,000件）を一括取得
- 口座名義のカナ正規化 → 完全一致 → Gemini AIによるファジー照合 の3段階で照合
- 照合結果をスプレッドシートのB列に書き込み（`照合済み（手配番号）` / `要確認` など）
- 一致した楽楽販売レコードの「銀振照合済み」フラグを「入金確認済み」に更新
- `--dry-run` オプションで書き込みなしの確認実行が可能

## 照合フロー

```
銀行振込名義（D列）
    │
    ▼
スキップ判定（法人・手数料・銀行振込等）
    │
    ▼
カナ名 正規化・抽出
    │
    ├─ 第1段階：正規化完全一致
    │       └─ 一致 → 照合済み
    │
    └─ 第2段階：先頭文字でグルーピング → Gemini AI照合
            ├─ 一致 → 照合済み
            ├─ 候補なし → 要確認（フリカナ未登録）
            └─ 不一致 → 要確認
```

## セットアップ

### 1. 依存パッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. 環境変数の設定

`.env.example` をコピーして `.env` を作成し、各値を設定します。

```bash
cp .env.example .env
```

| 変数名 | 説明 |
|--------|------|
| `RAKURAKU_TOKEN` | 楽楽販売 APIトークン（管理者設定 > メンテナンス機能タブ > APIパラメータ情報） |
| `GEMINI_API_KEY` | Google AI Studio で発行したGemini APIキー |
| `SPREADSHEET_ID` | 対象GoogleスプレッドシートのID |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | GoogleサービスアカウントのJSONをそのまま1行で記載 |
| `RAKURAKU_KINFU_ITEM_ID` | 楽楽販売の「銀振照合済み」項目ID |

### 3. スプレッドシートの形式

シート名は `YYYYMM`（例: `202604`）で、以下の列構成を想定しています。

| 列 | 内容 |
|----|------|
| B | 照合結果（ツールが書き込む） |
| C | 入金日 |
| D | 銀行口座名義（照合対象） |

## 使い方

```bash
# 当月シートで本番実行
python main.py

# 指定月シートで実行
python main.py --sheet 202603

# ドライラン（スプシ・楽楽への書き込みなし）
python main.py --dry-run

# 組み合わせ
python main.py --sheet 202603 --dry-run
```

├── main.py          # エントリポイント・照合ループ

├── matching.py      # カナ正規化・完全一致・Gemini照合ロジック

├── requirements.txt
